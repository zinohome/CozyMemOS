# GitHub 仓库设置指南

## 步骤 1: 在 GitHub 上创建新仓库

1. 登录 GitHub
2. 点击右上角的 "+" 号，选择 "New repository"
3. 填写仓库信息：
   - **Repository name**: `CozyMemOS` (或你喜欢的名称)
   - **Description**: `MemOS 1Panel 本地部署改造方案`
   - **Visibility**: 选择 Public 或 Private
   - **不要**勾选 "Initialize this repository with a README"（我们已经有了）
4. 点击 "Create repository"

## 步骤 2: 连接本地仓库到 GitHub

在终端中执行以下命令（将 `YOUR_USERNAME` 替换为你的 GitHub 用户名）：

```bash
# 添加远程仓库（替换 YOUR_USERNAME 为你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/CozyMemOS.git

# 或者使用 SSH（如果你配置了 SSH key）
# git remote add origin git@github.com:YOUR_USERNAME/CozyMemOS.git

# 查看远程仓库配置
git remote -v
```

## 步骤 3: 推送代码到 GitHub

```bash
# 推送主分支到 GitHub
git branch -M main
git push -u origin main
```

如果遇到认证问题，你可能需要：
- 使用 Personal Access Token（如果使用 HTTPS）
- 或者配置 SSH key（如果使用 SSH）

## 步骤 4: 验证

访问你的 GitHub 仓库页面，确认所有文件都已成功上传。

## 后续操作

### 添加更多提交

```bash
# 修改文件后
git add .
git commit -m "描述你的更改"
git push
```

### 创建分支

```bash
# 创建并切换到新分支
git checkout -b feature/your-feature-name

# 推送新分支
git push -u origin feature/your-feature-name
```

### 查看提交历史

```bash
git log --oneline
```

## 注意事项

- `project/` 目录已被 `.gitignore` 忽略，不会上传到 GitHub
- 确保不要提交敏感信息（API keys、密码等）
- 定期提交和推送你的更改
